# Dependency Update Pipeline

Trigger: `cve_alert` event, scheduled audit, or manual invocation.

## Stage 1: Scan

**Agent**: dependency-scanner (worktree isolation, Sonnet)
**On failure**: stop

Prompt:
```
Scan this project's dependencies for:
1. Known vulnerabilities (CVEs) — especially CRITICAL and HIGH
2. Packages more than 2 major versions behind
3. Deprecated packages with recommended replacements
4. License incompatibilities

Report with severity, package, current version, recommended version, and CVE ID if applicable.
```

## Stage 2: Impact Assessment

**Agent**: none (inline — orchestrator assesses breaking changes)
**On failure**: skip

From the scan results:

{{previous_output}}

For each recommended update:
1. Check if it's a major version bump (breaking changes likely)
2. Read the package's CHANGELOG or release notes if accessible
3. Check if any project code directly uses deprecated APIs
4. Classify: SAFE (patch/minor), REVIEW (major, likely safe), BREAKING (major, API changes)

Output: prioritized update plan with risk assessment.

## Stage 3: Update

**Agent**: remediation-agent (worktree isolation, Sonnet)
**On failure**: skip (partial updates are acceptable)

Prompt:
```
Apply the following dependency updates:

{{previous_output}}

Rules:
- Apply SAFE updates first (patches and minors)
- For REVIEW updates: apply the bump, check for compile/type errors
- For BREAKING updates: only apply if the impact assessment shows no API usage of removed features
- Update lockfiles after each batch
- Do NOT update devDependencies unless they have CVEs
```

## Stage 4: Test

**Agent**: test-runner (worktree isolation, Sonnet)
**On failure**: stop

Prompt:
```
Run full governance gates after dependency updates.
The following packages were updated:

{{previous_output}}

Run all gates. If any fail, identify which update caused the failure.
```

## Stage 5: PR

**Agent**: none (inline — orchestrator creates the PR)
**On failure**: skip

If Stage 4 passed:
```bash
git checkout -b deps/update-$(date +%Y%m%d)
git add -A
git commit -m "deps: update packages for security and freshness

{{stage1_cve_count}} CVEs resolved, {{stage3_update_count}} packages updated.

Co-Authored-By: Claude <noreply@anthropic.com>"
git push -u origin HEAD
gh pr create --title "deps: security and freshness updates" --body "## Dependency Updates

**Scan**: {{stage1_summary}}
**Updated**: {{stage3_summary}}
**Gates**: {{stage4_summary}}

### CVEs Resolved
{{stage1_cve_list}}

### Packages Updated
{{stage3_update_list}}

Generated by dependency-update pipeline."
```
