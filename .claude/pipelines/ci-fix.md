# CI Fix Pipeline

Trigger: `ci_failure` event from the pending_events queue.

## Stage 1: Diagnose

**Agent**: none (inline — orchestrator reads CI logs directly)
**On failure**: stop

Read the CI failure details from the event payload. Then:

```bash
gh run view <run_id> --log-failed 2>/dev/null | tail -100
```

Classify the failure:
- **Build error**: missing dependency, type error, syntax error
- **Test failure**: specific test case(s) failing
- **Lint/format**: style violations
- **Deploy error**: infrastructure issue
- **Flaky**: test passed on re-run (check if same test failed before)

Output: failure classification, root cause file(s), error messages.

## Stage 2: Fix

**Agent**: remediation-agent (worktree isolation)
**On failure**: stop

Prompt:
```
The CI pipeline failed. Here is the diagnosis:

{{previous_output}}

Apply a targeted fix. Rules:
- Only modify files directly related to the failure
- Do not refactor surrounding code
- If it's a test failure, fix the code (not the test) unless the test is wrong
- If it's a dependency issue, update the lockfile
- If it's a flaky test, add retry logic or stabilize the test
```

## Stage 3: Verify

**Agent**: test-runner (worktree isolation, Sonnet)
**On failure**: stop

Prompt:
```
Run the governance gates for this project to verify the fix.
The following fix was applied:

{{previous_output}}

Run all gates. Report pass/fail for each.
```

## Stage 4: PR

**Agent**: none (inline — orchestrator creates the PR)
**On failure**: skip

If Stage 3 passed:
```bash
git checkout -b fix/ci-$(date +%Y%m%d-%H%M%S)
git add -A
git commit -m "fix: resolve CI failure — {{stage1_classification}}

Automated fix via ci-fix pipeline.

Co-Authored-By: Claude <noreply@anthropic.com>"
git push -u origin HEAD
gh pr create --title "fix: resolve CI failure" --body "## Automated CI Fix

**Failure**: {{stage1_summary}}
**Root cause**: {{stage1_root_cause}}
**Fix applied**: {{stage2_summary}}
**Gates**: {{stage3_summary}}

Generated by ci-fix pipeline."
```
